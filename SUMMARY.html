End-of-Turn Detection

Plivo campus assignment 2026 · STT track · roll 2024ME10402 · repo plivo-ml-2024ME10402

1. Problem in one paragraph

A live voice agent must decide, at every silence ≥100 ms, whether the user is done speaking (eot) or only pausing mid-turn (hold). Answering too early interrupts the user; waiting too long adds dead air. We output a probability p_eot per pause. The official scorer sweeps a global (threshold, action-delay) pair and reports the lowest mean response delay achievable with ≤5% interrupted turns. Silence-only endpointing is the status quo (~1600 ms English / ~850 ms Hindi on this handout).

Why this beats silence

  • Silence treats every pause as EOT, so the only safe policy is a long timer.
  • Prosody ranks true ends above mid-turn holds → the sweep can pick a lower delay while staying under the 5% interrupt budget.
  • Missed EOTs cost a flat 1.6 s timeout; better ranking recovers those cases.

3. Method

Causality (hard requirement). For a pause at time t = pause_start, features use only audio[0:t], this pause’s index, and boundaries of earlier pauses in the same turn (they ended before t). The current row’s pause_end / duration / label are never inputs. Enforced in eot_dataset.build_matrix (a pause is appended to prior only after its own features are extracted) and documented in features_ext.py.

Features (v4 = v2 + literature cues, 60 dims)

  • Turn structure / energy / pitch / spectral: same causal v2 core (53 dims).
  • Cepstral flux: mean MFCC-frame Euclidean distance in the last ~400 ms + ratio vs prior window (lengthening → EOT).
  • Jitter / shimmer: period and amplitude perturbation on the last voiced run (creak / end-of-breath).
  • Semitone-z F0: last F0 and slope in semitones relative to context-median, z-scored in the causal context window.

Model

Soft 3-way blend: isotonic-calibrated ExtraTrees (w=0.30) + LogisticRegression C=1 (w=0.20) + isotonic-calibrated HistGradientBoosting (w=0.50). Calibration is nested inside GroupKFold. Chosen after 2-way blends (964 ms), v2 3-way (953.5 ms), pruning (942.5 ms), MLP/GRU, and feature ablations. Final checkpoint: model.joblib (+ blend_model.py). Inference CLI: 

python predict.py --data_dir <folder> --out predictions.csv

4. Iteration story (from RUNLOG)

  1. Silence baseline established the number to beat.
  2. v1 prosody + HGB cut mean delay to ~996 ms; LogReg was close but weaker AUC.
  3. HGB hyperparameter search did not improve the metric → stopped that line.
  4. Error dump: many low-p EOTs are first pauses of short turns → v2 early-turn features.
  5. ExtraTrees on v2 → 987.5 ms; iso-ET+LR blend → 964 ms.
  6. Hindi floor diagnosis; 3-way + HGB moved HI 850→784 on v2 (953.5 ms).
  7. Literature v4 cues + fine 3-way (0.30/0.20/0.50) → 912.2 ms (shipped).
  8. Prune/MLP/GRU/CNN: logged; none beat 7e before the ship window.

5. Remaining failures

  • Hindi long high-p HOLDs still bind the 5% budget, but calibrated HGB in the blend lowered HI delay from 850→792 ms.
  • Short complete answers with flat/rising pitch still look like holds (English).
  • Jitter/shimmer alone overfit; useful only inside the blend.
  • No lexical/ASR signal under the rules — pure acoustics.
